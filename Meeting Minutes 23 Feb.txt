📝 会议讨论内容
1. 项目进度与前端原型评估
* UI 设计审查：团队审查了使用 AI 工具生成的前端页面原型。讨论了界面的配色方案（蓝色与橙色的选择），并确认当前生成的页面在电脑端和手机端的适配效果。
* 组件开发：确认了前端按钮（Button）和卡片等元素的呈现方式，由于时间紧迫，决定直接借用已有的 UI 库组件并进行自定义修改，而非从零手写代码。
2. 技术架构与开发逻辑同步
为了解决技术落地的焦虑，Yin Jia 详细讲解了项目技术栈的运作逻辑：
* 数据库 (Database)：决定采用 Google 的 Firebase 替代过往项目使用的 MongoDB 作为云端数据库。探讨了数据的存储结构，如如何保存用户的进度（JSON格式存储）、分数的更新逻辑等。
* 后端逻辑 (Backend)：
   * 后端将使用 JavaScript 进行开发。
   * 接口与请求：通过生动的比喻（服务员与后厨）解释了前端（发送请求，如 80分）与后端（处理逻辑并连接数据库）的 API 通信原理。
   * 错误状态码 (Status Codes)：明确了前后端的报错判定标准。400 代表前端/客户端请求错误（如提交了空数据或格式错误），500 代表后端或服务器内部错误（如找不到数据库）。
* 前端逻辑 (Frontend)：强调了在前端代码中加入“等待 (Await)”和加载动画（Loading spinner）的重要性，以防在后端处理数据时前端页面报错或崩溃。
3. 资源支持与开发环境
* 云服务与 API：项目中需要用到 AI 进行数据分析，由于担心 API 调用的费用，Yin Jia 提出使用自己的 Google Cloud 账号，该账号含有免费额度（Credits）供团队调用，以解决资金问题。
* 代码托管：考虑到全员协作，Yin Jia 将负责在 GitHub 上创建分支（Branch）并配置权限，方便大家后续提交（Push）和拉取（Pull）代码。
